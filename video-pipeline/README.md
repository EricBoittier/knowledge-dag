# Video Pipeline (Linux + DaVinci MVP)

This project provides a low-risk, editor-first pipeline:
- normalize clips into DaVinci-friendly mezzanine media
- validate media streams before import
- generate a sequential FCPXML timeline

## Why this format

Default mezzanine policy:
- Container: `MOV`
- Video: `DNxHR HQ` (`dnxhd` + `dnxhr_hq`, `yuv422p`)
- Audio: `PCM s16le`, 48kHz stereo

This favors import reliability in DaVinci Resolve on Linux.

Optional review output:
- Container: `MP4`
- Video: `AV1` (`libsvtav1`)
- Audio: `PCM s16le`

## Repository layout

- `project_config.json`: pipeline config and media policy
- `src/media_probe.py`: ffprobe validation helpers
- `src/normalize_clips.py`: clip normalization + manifest writing
- `src/build_timeline_fcpxml.py`: sequential FCPXML generation
- `src/run_pipeline.py`: one-command orchestration
- `src/overlay_manifest.py`: overlay/subtitle schema validation
- `src/subtitle_builder.py`: `.ass` + `.srt` generation
- `src/overlay_scheduler.py`: sentence-timed image event scheduling
- `src/compose_overlay_preview.py`: FFmpeg preview composition
- `scripts/test_import_check.sh`: pre-import stream checks
- `scripts/copy_obrainrot_assets.sh`: sparse-sync OBrainRot assets + provenance
- `templates/timeline_base.xml`: optional XML template notes
- `output/`: generated files

## Requirements

- Python 3.10+
- `ffmpeg` and `ffprobe` available in PATH

## Quick start

1. Put source videos in `./input` (or override via `--input-dir`).
2. Run:

```bash
python3 src/run_pipeline.py --config ./project_config.json
```

For a more visually edited demo timeline (reordered + trimmed clips):

```bash
python3 src/run_pipeline.py --config ./project_config.json --showcase
```

Generate subtitle/overlay artifacts plus a preview render:

```bash
python3 src/run_pipeline.py --config ./project_config.json --showcase --with-overlays --with-preview
```

Use Gemini-generated script/edit metadata for a richer timeline package:

```bash
python3 src/run_pipeline.py \
  --config ./project_config.json \
  --showcase \
  --with-overlays \
  --with-preview \
  --timeline-variants horizontal,vertical \
  --gemini-project-dir ../ai-director-app/projects/walrus-dfs
```

Generate multi-character dialogue audio from checkpoint-cycled CSM voices:

```bash
python3 src/run_pipeline.py \
  --config ./project_config.json \
  --showcase \
  --with-overlays \
  --gemini-project-dir ../ai-director-app/projects/walrus-dfs \
  --generate-dialogue-audio
```

Prebake overlays and colored text directly into normalized clips (DaVinci-safe) for both orientations:

```bash
python3 src/run_pipeline.py \
  --config ./project_config.json \
  --showcase \
  --with-overlays \
  --gemini-project-dir ../ai-director-app/projects/walrus-dfs \
  --timeline-variants horizontal,vertical \
  --prebake-overlays
```

To remove stale normalized/review files before regenerating:

```bash
python3 src/run_pipeline.py --config ./project_config.json --force --clean-old-normalized --showcase
```

Outputs:
- normalized clips in `output/normalized`
- manifest at `output/normalized/manifest.json`
- timeline at `output/timeline_davinci_resolve.fcpxml`
- showcase manifest (when enabled): `output/normalized/manifest.showcase.json`
- overlays directory (when enabled): `output/overlays/`
  - `subtitles.ass`
  - `subtitles.srt`
  - `image_events.json`
  - `overlay_manifest.json`
  - `dialogue_plan.json` (when `--gemini-project-dir` is used)
  - `dialogue_mix.wav` (when `--generate-dialogue-audio`)
  - `preview_overlay.mp4` (when `--with-preview`)
- timeline variants:
  - `output/timeline_davinci_resolve.horizontal.fcpxml`
  - `output/timeline_davinci_resolve.vertical.fcpxml`
- prebaked clips (when `--prebake-overlays`):
  - `output/prebaked/horizontal/*.baked.mov`
  - `output/prebaked/vertical/*.baked.mov`
  - per-variant prebaked manifests for timeline asset mapping

## Explicit policy ffmpeg commands

### Normalize to DaVinci-safe MOV (DNxHR + PCM)

```bash
ffmpeg -y -i INPUT \
  -map 0:v:0 -map 0:a:0 \
  -c:v dnxhd -profile:v dnxhr_hq -pix_fmt yuv422p -r 30 \
  -c:a pcm_s16le -ar 48000 -ac 2 \
  OUTPUT.mov
```

### Optional review file (AV1 + PCM)

```bash
ffmpeg -y -i INPUT \
  -map 0:v:0 -map 0:a:0 \
  -c:v libsvtav1 -crf 30 -preset 6 \
  -c:a pcm_s16le \
  OUTPUT.review.mp4
```

## DaVinci import flow

1. Import normalized media first.
2. Import `output/timeline_davinci_resolve.fcpxml`.
3. If media relinking is needed, use absolute paths generated by this pipeline.

## Overlay feature flags

- `--with-overlays`: generate subtitle files and overlay manifest
- `--with-preview`: render a fast H.264 preview with burned subtitles and image overlays
- `--gemini-project-dir`: use `script-lines.json` + `edit-annotations.json` to create style-rich overlays
- `--timeline-variants horizontal,vertical`: emit both orientation timelines
- `--prebake-overlays`: burn image overlays + colored text into per-clip DNxHR outputs for reliable DaVinci import
- `--generate-dialogue-audio`: synthesize multi-character WAV from dialogue plan and checkpoint cycle
- Overlay markers from `overlay_manifest.json` are added as clip markers in generated FCPXML

## Local B-roll analyzer (image-to-text)

The normalization stage can run a local image-to-text model against sampled frames and write
ranked B-roll windows into the manifest.

Config in `project_config.json`:

- `broll_analyzer.enabled`: enable/disable analysis
- `broll_analyzer.model_name`: local Hugging Face model id
- `broll_analyzer.device`: `cpu` or `cuda`
- `broll_analyzer.sample_interval_sec`: frame sampling interval
- `broll_analyzer.window_duration_sec`: clip window duration proposal
- `broll_analyzer.max_windows`: number of retained candidates
- `broll_analyzer.min_window_score`: quality threshold
- `timeline.use_broll_top_window`: auto-trim timeline clips to top-ranked B-roll window

Manifest fields per entry:

- `broll_analysis`: full analysis payload (schema + model + reason)
- `broll_windows`: ranked candidate windows
- `broll_top_window`: best window for auto-trim
- `broll_markers`: editorial marker suggestions

## Gemini B-roll fit judge

After normalization, an optional Gemini pass can evaluate each clip against project context
and decide whether it should remain in the b-roll timeline.

Config (`project_config.json`):

- `gemini_broll_judge.enabled`
- `gemini_broll_judge.model`
- `gemini_broll_judge.min_fit_score`
- `gemini_broll_judge.max_windows_in_prompt`
- `gemini_broll_judge.apply_to_timeline`

Environment:

- `GEMINI_API_KEY` must be set when judge is enabled.

Outputs (per manifest entry):

- `broll_judgement.include` (final include/exclude)
- `broll_judgement.fit_score` (0..1)
- `broll_judgement.reason`

When `apply_to_timeline` is true, `timeline.enabled` is updated based on the Gemini decision.

## Dialogue chunking for weak voice checkpoints

- Dialogue synthesis uses short chunks (`max_words_per_chunk`, default `5`) before each CSM generation call.
- Sentence boundaries use NLTK (`punkt`) when available, with regex fallback when not.
- Configure in `project_config.json` under `overlays.dialogue.max_words_per_chunk`.

## Config-driven style theme mapping (Gemini annotations)

Use `overlays.style_theme_rules` to map annotation tokens to visual profiles without changing code.
Each rule has:
- `profile`: one of `default`, `shorts`, `tiktok`, `dialogue`
- `keywords`: matched against Gemini `effects`, `transition`, and `lut_hint`

The first matching rule wins, so place more specific rules earlier.

## OBrainRot asset sync and provenance

Sync upstream assets into local input storage:

```bash
./scripts/copy_obrainrot_assets.sh
```

Pin to a specific upstream commit for reproducibility:

```bash
PINNED_COMMIT=<commit_sha> ./scripts/copy_obrainrot_assets.sh
```

Each sync writes a provenance file at:
- `input/assets/obrainrot/.source-provenance.json`

This records source repository, selected ref, resolved commit SHA, and sync timestamp.

## Experimental: local style transfer on video

You can run offline style transfer over a clip by extracting frames, processing each frame
with your local image model (`z-image`), and re-encoding with source audio.

Script:
- `scripts/style_transfer_video.py`

Example:

```bash
python3 scripts/style_transfer_video.py \
  --input ./input/sample.mp4 \
  --output ./output/style_transfer/sample.stylized.mp4 \
  --fps 12 \
  --overwrite \
  --z-image-cmd 'z-image stylize --input "{input}" --output "{output}" --style "{style}" --prompt "{prompt}" --negative "{negative_prompt}"' \
  --style-image ./input/style_refs/ink.jpg \
  --prompt "high-contrast manga ink, cinematic shadows" \
  --negative-prompt "blurry, low detail, watermark"
```

Notes:
- `--z-image-cmd` is a template and must include `{input}` and `{output}` placeholders.
- Optional placeholders: `{style}`, `{prompt}`, `{negative_prompt}`.
- Keep `--fps` modest first (for example `8-12`) to control runtime and reduce flicker risk.

Faster mode (stylize fewer frames, then interpolate):

```bash
python3 scripts/style_transfer_video.py \
  --input ./input/sample.mp4 \
  --output ./output/style_transfer/sample.stylized.fast.mp4 \
  --fps 24 \
  --stylize-every-nth-frame 3 \
  --interpolate-mode minterpolate \
  --scene-cut-aware \
  --overwrite \
  --z-image-cmd 'z-image stylize --input "{input}" --output "{output}" --style "{style}" --prompt "{prompt}"' \
  --style-image ./input/style_refs/ink.jpg
```

Interpolation modes when `--stylize-every-nth-frame > 1`:
- `none`: duplicate frames (fastest, choppier)
- `framerate`: blend interpolation (fast, can ghost)
- `minterpolate`: motion-compensated interpolation (slowest, usually best motion)

Scene-cut options:
- `--scene-cut-aware`: runs interpolation separately per detected scene segment
- `--scene-threshold`: scene sensitivity (default `0.3`); lower values detect more cuts
